# Environment Variables Reference

This guide provides a comprehensive reference for all environment variables used to configure Skyvern.

## Quick Setup

Generate a `.env` file automatically:

```bash
skyvern init llm
```

This interactive CLI will guide you through configuring your LLM provider and other essential settings.

## Core Configuration

### Application Settings

| Variable | Description | Type | Default | Required |
|----------|-------------|------|---------|----------|
| `ENV` | Environment name (development, staging, production) | String | `development` | No |
| `LOG_LEVEL` | Logging level | String | `INFO` | No |
| `LOG_FORMAT` | Log output format | String | `json` | No |
| `SKYVERN_DEBUG` | Enable debug mode | Boolean | `false` | No |
| `PORT` | API server port | Integer | `8000` | No |
| `HOST` | API server host | String | `0.0.0.0` | No |
| `ALLOWED_ORIGINS` | CORS allowed origins (comma-separated) | String | `*` | No |

**Example:**
```bash
ENV=production
LOG_LEVEL=INFO
LOG_FORMAT=json
PORT=8000
HOST=0.0.0.0
ALLOWED_ORIGINS=https://app.skyvern.com,https://admin.skyvern.com
```

### API Configuration

| Variable | Description | Type | Default | Required |
|----------|-------------|------|---------|----------|
| `API_KEY` | Default API key for authentication | String | - | No |
| `ENABLE_API_KEY_AUTH` | Require API key authentication | Boolean | `true` | No |
| `API_KEY_HEADER` | Header name for API key | String | `x-api-key` | No |
| `MAX_REQUEST_SIZE` | Maximum request body size (bytes) | Integer | `10485760` | No |
| `REQUEST_TIMEOUT` | Request timeout (seconds) | Integer | `300` | No |

**Example:**
```bash
ENABLE_API_KEY_AUTH=true
API_KEY_HEADER=x-api-key
MAX_REQUEST_SIZE=20971520  # 20MB
REQUEST_TIMEOUT=600  # 10 minutes
```

## LLM Provider Configuration

### OpenAI

| Variable | Description | Type | Required |
|----------|-------------|------|----------|
| `ENABLE_OPENAI` | Register OpenAI models | Boolean | No |
| `OPENAI_API_KEY` | OpenAI API key | String | Yes (if enabled) |
| `OPENAI_API_BASE` | Custom API base URL | String | No |
| `OPENAI_ORGANIZATION` | Organization ID | String | No |

**Recommended Models:**
- `OPENAI_GPT4O` - GPT-4o (best performance)
- `OPENAI_GPT4O_MINI` - GPT-4o-mini (faster, cheaper)
- `OPENAI_O1_MINI` - O1-mini (reasoning tasks)
- `OPENAI_O3` - O3 (advanced reasoning)

**Example:**
```bash
ENABLE_OPENAI=true
OPENAI_API_KEY=sk-proj-xxx
LLM_KEY=OPENAI_GPT4O
SECONDARY_LLM_KEY=OPENAI_GPT4O_MINI
```

|----------|-------------|------|----------|
| `ENABLE_ANTHROPIC` | Register Anthropic models | Boolean | No |

```bash
LLM_KEY=ANTHROPIC_CLAUDE4_SONNET
SECONDARY_LLM_KEY=ANTHROPIC_CLAUDE3.5_SONNET
```

### Primary and Secondary LLM

| Variable | Description | Type | Required |
|----------|-------------|------|----------|
| `LLM_KEY` | Primary LLM for heavy tasks | String | Yes |
| `SECONDARY_LLM_KEY` | Secondary LLM for light tasks | String | No |

**Best Practices:**
- Use powerful models (GPT-4o, Claude 4 Sonnet) for primary LLM
- Use faster/cheaper models (GPT-4o-mini, Claude 3.5 Sonnet) for secondary LLM

**Example:**
```bash
LLM_KEY=ANTHROPIC_CLAUDE4_SONNET
SECONDARY_LLM_KEY=ANTHROPIC_CLAUDE3.5_SONNET
```

### S3-Compatible Storage

| Variable | Description | Type | Default | Required |
|----------|-------------|------|---------|----------|
| `ARTIFACT_STORAGE_PATH` | Local storage path (if not using S3) | String | `./artifacts` | No |

```bash
AWS_ACCESS_KEY_ID=AKIAIOSFODNN7EXAMPLE
AWS_SECRET_ACCESS_KEY=wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY
```

**Example (MinIO):**
```bash
AWS_S3_BUCKET_ARTIFACTS=skyvern-artifacts
S3_ENDPOINT_URL=http://minio:9000
AWS_ACCESS_KEY_ID=minioadmin
AWS_SECRET_ACCESS_KEY=minioadmin
```

## Browser Configuration

### Browser Settings

| Variable | Description | Type | Default | Required |
|----------|-------------|------|---------|----------|
| `BROWSER_TYPE` | Browser type | String | `chromium-headless` | No |
| `BROWSER_TIMEOUT` | Browser timeout (ms) | Integer | `30000` | No |
| `MAX_SCRAPING_RETRIES` | Max retry attempts | Integer | `3` | No |
| `VIDEO_PATH` | Path for browser recordings | String | - | No |
| `BROWSER_ACTION_TIMEOUT_MS` | Action timeout (ms) | Integer | `5000` | No |

**Browser Types:**
- `chromium-headless` - Headless Chrome (default)
- `chromium` - Chrome with UI
- `firefox-headless` - Headless Firefox
- `firefox` - Firefox with UI
- `webkit-headless` - Headless WebKit
- `webkit` - WebKit with UI
- `cdp-connect` - Connect to existing Chrome via CDP

**Example:**
```bash
BROWSER_TYPE=chromium-headless
BROWSER_TIMEOUT=60000  # 60 seconds
MAX_SCRAPING_RETRIES=5
```

### CDP Connect Mode

| Variable | Description | Type | Required |
|----------|-------------|------|----------|
| `CHROME_EXECUTABLE_PATH` | Path to Chrome executable | String | No |
| `CDP_URL` | Chrome DevTools Protocol URL | String | No |

**Example (Local Chrome):**
```bash
BROWSER_TYPE=cdp-connect
CHROME_EXECUTABLE_PATH="/Applications/Google Chrome.app/Contents/MacOS/Google Chrome"
```

**Example (Remote Chrome):**
```bash
BROWSER_TYPE=cdp-connect
CDP_URL=ws://chrome:9222/devtools/browser/xxx
```

## Security Best Practices

1. **Never commit `.env` files** to version control
2. **Use secret managers** in production (AWS Secrets Manager, Azure Key Vault, etc.)
3. **Rotate credentials regularly**
4. **Use minimal permissions** for database and cloud access
5. **Enable TLS/SSL** for all network connections
6. **Set strong API keys** (minimum 32 characters)
7. **Restrict CORS origins** in production

## Configuration Validation

Validate your configuration:

```bash
skyvern validate-config
```

This checks:
- Required variables are set
- Database connectivity
- LLM provider configuration
- Storage accessibility
- Redis connectivity (if configured)
